```

설계 결정 (W3와 동일):
1. 산출물 형태: 운영 SOP 체크리스트 + 시범 큐레이션 1회분

3. Drive 자료 가져오기: 참조만 (메타데이터 + 링크)
4. 편집자 승인 게이트: 2단계 (digest + 발행 직전)

저장 위치: reports/monthly_digest_2026-04-W4.md

W3와 비교 분석 추가 요구:
- W4 신규 클러스터 (W3에 없던 주제군)
- W3 클러스터의 W4 연속성 (예: bedrock-permission-403이 W4도 등장하는지)
- W3 갭 분석 5건 중 W4에 일부라도 다뤄진 영역이 있는지

source_id 명명 규칙은 docs/integrations/sns_to_magazine_pipeline.md §1 따라:
- sns-blog-YYYYMMDD-<slug>
- sns-cardnews-YYYYMMDD[-NN]

source_registry_status: proposed (Gate 1 미승인 상태로 작성)

Gate 2 사전 체크 6건 자동 첨부 (W3 디제스트 §4 동일).
AI 사용 고지 명시.

W3 디제스트는 reports/monthly_digest_2026-04-W3.md 참조.
SOP는 docs/integrations/sns_to_magazine_pipeline.md 참조.
```

## 예상 결과 검증 항목

Cowork이 W4 디제스트 생성 후 사용자가 확인할 5가지:

### 1. 형식 일관성
- [ ] W3와 동일한 헤더 (대상 기간·입력 통계·처리 모드·SOP 링크·source_registry_status)
- [ ] `editor_approval` YAML 블록 (status: pending)
- [ ] 7일치 입력 인벤토리 (주말 미가동 확인)
- [ ] 클러스터 섹션 (Cluster A·B·...·N)
- [ ] 카드뉴스 5덱 매핑 표
- [ ] 갭 분석 표
- [ ] Gate 2 사전 체크 6건
- [ ] AI 사용 고지

### 2. W3와의 비교 분석 (신규 추가 요구사항)
- [ ] W3 5 클러스터(A~E) 중 W4 연속 등장 클러스터 명시
- [ ] W4 신규 클러스터 (W3에 없던 주제) 별도 표기
- [ ] 슬러그 반복 패턴 변화 (W3에서 5회였던 bedrock-permission-403이 W4에서 몇 회?)

### 3. 갭 분석 추세
- [ ] W3 갭 5건 중 W4에 다뤄진 영역 있는지
- [ ] W4 신규 갭 영역 발견되는지
- [ ] backlog의 SNS 디제스트 갭 분석 항목과 정합 (중복·확장)

### 4. SOP 정합성
- [ ] 메타데이터만 보유 (본문 무복사)
- [ ] source_id 명명 규칙 준수
- [ ] rights_status 'free' 가정 (또는 편집자 확인 필요 표기)

### 5. 정합 fix 자동 적용 (W3에서 확정)
- [ ] `source_registry_status: proposed` 헤더 (W3에서 정합 fix로 추가)
- [ ] Gate 2 체크 6번 (카드뉴스 매핑 검증)

## 활용 시나리오

W4 디제스트가 W3와 비슷한 패턴이면:
- **5월 매거진 콘텐츠 풀 = W3 + W4 통합** → 월말 종합 디제스트(`reports/monthly_digest_2026-04.md`) 작성 가능
- 동일 클러스터 8~10회 반복은 **운영 트러블슈팅 시리즈** 본격 기획 신호
- 갭 영역이 2주 연속 0건이면 **갭 항목 전체를 5월 plan_issue로 우선 승격**

W4가 W3와 크게 다르면:
- 자매 시스템 운영 변경 또는 주제 다변화 신호 — 편집자가 SOP §5 캘린더 cadence 재검토
- 클러스터링 임계값(현재: 동시출현 ≥2) 조정 검토 가능
